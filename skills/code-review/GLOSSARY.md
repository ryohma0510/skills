# 症状ラベルの定義

SKILL.md の各 Check が使う症状ラベルの定義。「Fake Generalization」は APoSD 本文にはない、本スキル独自の用語。

## Change Amplification

- 定義：一見単純な変更のために、多くの場所を書き換える必要がある状態。コピペされた定数・重複した知識が典型。
- 使用箇所：Check 1（Implementation Leakage）・Check 4（Code-Repetition Comment。コメントがコードの操作を言い換えているだけだと、実装を変えるたびにコメントも書き直す必要が生じる）

## Information Leakage

- 定義：同じ設計上の知識が複数のモジュールに現れている状態。コードの重複はその強いシグナル。片方を変えるともう片方も直す必要がある＝変更の増幅の温床。
- 使用箇所：Check 2（Vocabulary Leakage）

## Cognitive Load

- 定義：タスクを完了するために開発者が知っていなければならないことの量。コードの行数とは別物——行数が増えても知るべきことが減るなら、それは単純化。
- 使用箇所：Check 2・Check 3・Check 4（コメントが操作の言い換えに留まり対象の意味を語らないと、読み手はその意味を自分で推論するしかなくなる）

## Fake Generalization

- 定義：一つのユースケースでしか使わない関心事を、nullable／デフォルト値つき引数として汎用的に見せかけて持たせること。一見汎用化された設計に見えるが、他の呼び手には「このパラメータは何のためのものか」という余計な認知的負荷を生む。
- 使用箇所：Check 3（One-Sided Argument、低確信度）
